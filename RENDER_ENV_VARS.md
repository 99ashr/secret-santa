## Required Variables

### 1. **SECRET_KEY** (Most Important!)

**What it is:** A random secret string that Django uses to encrypt sessions and tokens.

**Where to get it:**

Run this command in your terminal:
```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

It will print something like:
```
k&x%jn9f#z@x2q!r$9v*w+z^a3b&4c5d6e7f8g9h0i1j2k3l4
```

**In Render:**
- **Key:** `SECRET_KEY`
- **Value:** (paste the long string from above)

Example:
```
Key:   SECRET_KEY
Value: k&x%jn9f#z@x2q!r$9v*w+z^a3b&4c5d6e7f8g9h0i1j2k3l4
```

---

### 2. **DEBUG**

**What it is:** Tells Django to show detailed error messages or hide them (production should hide them).

**In Render:**
- **Key:** `DEBUG`
- **Value:** `False`

```
Key:   DEBUG
Value: False
```

---

### 3. **ALLOWED_HOSTS**

**What it is:** Tells Django which domains can access your app.

**In Render:**
- **Key:** `ALLOWED_HOSTS`
- **Value:** `*` (allows all domains; safe for this app)

Or, if you want to be specific (replace `secret-santa-xxxxx` with your actual Render app name):
```
Key:   ALLOWED_HOSTS
Value: secret-santa-xxxxx.onrender.com,localhost,127.0.0.1
```

For simplicity, use:
```
Key:   ALLOWED_HOSTS
Value: *
```

---

## Optional Variables (Usually Not Needed)

### 4. **DATABASE_URL** (Auto-generated by Render ‚Äî Don't add manually)

Render automatically creates this when your web service deploys. You don't need to add it manually. It will look like:
```
postgresql://user:password@host:port/dbname
```

If your database doesn't connect after deployment, this might not be set. In that case, Render support can help.

## How to Add Them in Render

1. In Render Dashboard, click your **"secret-santa"** service
2. Scroll down to **"Environment Variables"** section
3. Click **"Add Environment Variable"** (or similar button)
4. For each variable:
   - Enter **Key** (e.g., `SECRET_KEY`)
   - Enter **Value** (e.g., the long secret string)
   - Click **"Add"** or **"Save"**
5. After adding all three, click **"Save & Deploy"** or **"Deploy"** button
6. Render will restart your app with the new variables

## Troubleshooting

### Problem: "SECRET_KEY is not set"
- Make sure you ran the command above and got a long random string
- Copy the entire string (don't leave off any characters)
- Paste it into Render exactly as-is

### Problem: "ALLOWED_HOSTS error"
- Make sure value is exactly: `*` (asterisk only)
- Or include your Render domain: `secret-santa-xxxxx.onrender.com`

### Problem: "DEBUG is still True in production"
- Make sure DEBUG value is exactly: `False` (capital F, lowercase alse)
- Not `false`, not `True`, not `yes` ‚Äî exactly `False`

### Problem: App won't start after adding variables
- Check Render's "Logs" tab to see error messages
- Common issues:
  - SECRET_KEY has special characters that need escaping (use the generated one as-is)
  - ALLOWED_HOSTS format is wrong
  - Missing equals sign or quotes somewhere

## Extra: Where These Variables Are Used

In your Django code (`secret_santa_config/settings.py`):

```python
from decouple import config

SECRET_KEY = config('SECRET_KEY', default='...')
DEBUG = config('DEBUG', default=True, cast=bool)
ALLOWED_HOSTS = config('ALLOWED_HOSTS', default='localhost,127.0.0.1').split(',')
```

Django reads these environment variables automatically!
